# ⚡ elcView — עוזר ביקורת לחשמלאים

אפליקציית ווב (HTML + JavaScript, ללא build) לחשמלאים: מעלים PDF של תכנית חשמל,
המערכת מזהה את השקעים והחדרים, ומפיקה טבלת ביקורת עם ייצוא לאקסל.

## יכולות (שלב א')

- **העלאת שרטוט PDF** — רינדור עם pdf.js, זום, גרירה ותמיכה במגע.
- **זיהוי שקעים אוטומטי** — תוויות הגובה (`H=60`, `H=120`...) וסימוני `TV`/`T`
  נקראים משכבת הטקסט ומההערות המוטמעות של ה-PDF (ייצוא אוטוקאד) עם מיקום מדויק.
- **זיהוי חדרים ודאי מההערות** — ייצוא אוטוקאד מטמיע את טקסט השרטוט
  כהערות PDF בקידוד מקלדת ('nycj' = מטבח); המערכת מפענחת אותן ומזהה את
  שמות החדרים, המרחקים ומספרי המעגלים בלי OCR. בנוסף, המשתמש יכול להוסיף
  הערות טקסט (FreeText) עם שמות חדרים בכל עורך PDF — והן נקראות בוודאות
  מלאה, כולל שמות שאינם במילון.
- **זיהוי חדרים ב-OCR (גיבוי)** — ל-PDF בלי הערות: Tesseract.js בעברית
  מציע שמות חדרים ומרחקים; המשתמש מותח את מלבני החדרים לגבולות האמיתיים.
- **🤖 ניתוח Claude** — כפתור במסך השרטוט ששולח את תמונת השרטוט + כל
  הטקסטים שחולצו ל-Claude (Opus 4.8) דרך Supabase Edge Function, ומקבל
  ניתוח מלא: חדרים עם גבולות אמיתיים, נקודות עם גובה/כמות/מרחק/מעגל.
  דורש הגדרת הסוד `ANTHROPIC_API_KEY` ב-Supabase (Edge Functions → Secrets).
  קוד הפונקציה: `supabase/functions/analyze-plan/index.ts`.
- **עריכה על גבי השרטוט** — הוספה/גרירה/מחיקה של שקעים וחדרים, גובה מהרצפה,
  מרחק מפינת הקיר, מעגל וסוג נקודה.
- **טבלת ביקורת** — מקובצת לפי חדר; עריכה מותרת רק בעמודות "בוצע" ו"הערות"
  (נשמר אוטומטית); ייצוא לקובץ אקסל RTL.
- **בסיס נתונים** — Supabase (פרויקטים, חדרים, שקעים + Storage לשרטוטים).

## יכולות (שלבים ב'–ד': ביקורת בשטח עם מצלמה)

- **זיהוי קופסה כתומה** — סינון צבע HSV + רכיבים קשירים על פריים המצלמה;
  קנה המידה נגזר מהגודל התקני של קופסת השחלה (55 מ"מ).
- **הצבעה על שקע ⇒ מידות** — הקשה על קופסה מזוהה מקפיאה את הפריים ומודדת
  גובה מהרצפה (עד קו הרצפה) ומרחק מפינת הקיר. אם זיהוי הרצפה/פינה טעה —
  מקישים ידנית על הנקודה הנכונה.
- **השוואה לשרטוט** — המדידה מותאמת לנקודה המתאימה בחדר ומסומנת ✓/✗
  לפי סטייה מותרת (ברירת מחדל 5 ס"מ, ניתן לשינוי).
- **סריקה אוטומטית** — במצב וידאו רציף, קופסה שמזוהה יציב נמדדת, מושווית
  ומסומנת "בוצע" אוטומטית כשהיא תואמת.
- המדידות נשמרות ב-DB ומופיעות בטבלה ובקובץ האקסל (גובה נמדד / מרחק נמדד / תואם שרטוט).

> דיוק המדידה תלוי בצילום ניצב לקיר ובזיהוי נכון של קו הרצפה — מומלץ
> לוודא שהקופסה וקו הרצפה שמתחתיה נראים יחד בפריים.

## הרצה

```bash
python -m http.server 8123
# ולפתוח http://localhost:8123
```

## מבנה הקוד

```
index.html            מסכים: פרויקטים / שרטוט / טבלה / ביקורת
css/style.css         עיצוב RTL, מותאם למובייל
libs/                 pdf.js מקומי (worker חייב להיות same-origin)
js/
  app.js              בקר ראשי (class App)
  config.js           הגדרות: Supabase, ראייה ממוחשבת, מילון חדרים
  models.js           Project / Room / Outlet
  services/
    pdfService.js     טעינה ורינדור PDF, חילוץ טקסט (pdf.js)
    detector.js       מנוע הזיהוי: H=xx ⇒ שקעים, שיוך לחדרים
    ocrService.js     זיהוי שמות חדרים (Tesseract.js heb, PSM 11)
    vision.js         זיהוי קופסאות כתומות (HSV + רכיבים קשירים)
    measure.js        מנוע המדידה: פיקסלים ⇒ ס"מ, קו רצפה ופינת קיר
    db.js             Supabase: CRUD + Storage
    excelExporter.js  ייצוא xlsx (SheetJS)
  ui/
    planViewer.js     קנבס + שכבת עריכה אינטראקטיבית (זום/פאן/גרירה)
    tableView.js      טבלת הסיכום
    inspectionView.js מסך ביקורת: מצלמה, מדידה בהצבעה, השוואה וסריקה אוטומטית
```

## שימוש בשטח (מהטלפון)

1. פתח את האתר (GitHub Pages, HTTPS) ובחר את הפרויקט.
2. בטאב "ביקורת" בחר את החדר שבו אתה נמצא ופתח את המצלמה.
3. כוון אל קופסה כתומה — היא תסומן במסגרת. הקש עליה לקבלת המידות,
   או הפעל "סריקה אוטומטית" ועבור עם המצלמה על הקירות.
